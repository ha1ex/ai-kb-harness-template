# 5-minute rule — agent dev-day estimate → 5 min reality

> Problem solved: Founders take coding-agent time estimates literally ("4 days for the auth refactor") and over-allocate calendar; the actual ratio in agent-driven vibe-coding is roughly 5 minutes per estimated day.

## What

Heuristic: T_real ≈ 5 × D_estimated (minutes per day). A "4-day infrastructure setup" is ~20 min of focused vibe-coding. Use this to right-size planning sessions and stop padding sprints around inflated agent estimates.

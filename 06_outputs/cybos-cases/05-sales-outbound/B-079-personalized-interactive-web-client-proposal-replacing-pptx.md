# Personalized interactive web client proposal (replacing PPTX portfolio)

> Consultants build 40-slide PowerPoint decks per client over 4-6 hours. An interactive web page generated in 5 minutes tests better.

## What

Replace the static PPT "portfolio proposal" with a per-client interactive web page generated from that client's history. Each page pulls the client's risk profile, holdings, recent conversations, life events from the CRM, and renders a single-page web artifact: scenario sliders, drill-downs, embedded charts, narrative annotations in the relationship manager's voice. Generated by a Claude Code skill; embedded in the consultant's existing workflow tool.

## Why it matters

At one large investment firm, the previous deliverable was a 40-slide clunky PowerPoint that consultants assembled by hand in 4–6 hours. The interactive web version is generated in under 5 minutes per client and tested better in client meetings: clients interact with sliders during the call rather than nodding through slides. The firm cascaded the skill to all consultants within 3 weeks.

## End-to-end

1. Templating phase: design the canonical client page (HTML + Tailwind) with 6–8 named slots: profile band, scenario panel, holdings table, life-events sidebar, narrative paragraphs, CTA.
2. Build a Claude Code skill `client-proposal {ClientID}` that pulls CRM rows + last 12 months of meeting notes for that client.
3. Skill body: render the template with the client's data, plus three LLM-written narrative blocks anchored on the consultant's voice doc.
4. Deploy as a static site or one HTML file per client (signed URL, expires in 30 days).
5. Wire into the consultant workflow tool: a `Generate proposal` button → triggers the skill → returns a URL.
6. Internal review gate before client-send: a junior reviewer eyeballs the narrative blocks for any fabricated numbers.
7. Track outcomes (meeting → engagement → conversion) per generated proposal.

## Gotchas

- LLM-written numbers are the enemy: render every number from the structured CRM read, not from the LLM. The LLM only writes narrative paragraphs that the structured numbers anchor to. The internal review gate exists for the day someone forgets this rule.

## Tools

- Claude Code skill harness
- CRM with per-client transcript / holdings access
- Static-host target (Netlify / Vercel / internal S3)
- A `voice_doc.md` per consultant or per RM team
